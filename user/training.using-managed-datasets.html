我们建议您使用 Vertex AI 代管式数据集来训练自定义模型，而不是编写训练应用来直接提取训练数据。

对象数据集#

以下是 s3://aip/schema/dataset/ioformat/json_no_specific_target_io_format_1.0.0.yaml 里的定义。

title: JsonNoSpecificTarget
description: >
  Import and export format for importing/exporting text together without specific
  target. Can be used in Dataset.import_schema_uri field.
type: object
required:
- content
properties:
  content:
    oneOf:
    - type: object
      properties:
        type:
          type: string
          enum: [jsonContent]
        jsonContent:
          type: json
          description: Full length json content. Up to 10MB in size.
    - type: object
      properties:
        type:
          type: string
          enum: [jsonS3Uri]
        jsonS3Uri:
          type: string
          description: >
            A Cloud Storage URI pointing to a json file. Up to 10MB in
            size. Supported file mime types: `application/json`.
    discriminator:
      propertyName: type
  languageCode:
    type: string
    description: Optional language code of the text.
    default: en-US
  dataItemResourceLabels:
    description: >
      Resource labels on the DataItem. Overrides values set in
      ImportDataConfig at import time. Can set a user-defined label
      or the predefined `aip.mesoor.com/ml_use` label, which is
      used to determine the data split and can be set to `training`, `test`,
      and `validation`.
    type: object
    additionalProperties:
      type: string

以下示例展示了如何使用定义创建自己的 JSON 行文件。

{
  "jsonContent": {
    "id": 42,
    "keys..": "values.."
  }
}
{
  "jsonS3Uri": "s3_uri_to_file",
  "dataItemResourceLabels": {
    "aiplatform.googleapis.com/ml_use": "training|test|validation"
  }
}

第一个文档包含一个必要的 jsonContent 字段。

第二个文档包含一个必要的 jsonS3Uri 和可选的 dataItemResourceLabels 字段。

为了便于阅读，以上示例包括换行符。 在 JSON 文件中，请仅在每个文档后面添加换行符，实际文件如以下示例。

{"jsonContent":{"id":42,"keys..":"values.."}}
{"jsonS3Uri":"s3_uri_to_file","dataItemResourceLabels":{"aiplatform.googleapis.com/ml_use":"training|test|validation"}}
